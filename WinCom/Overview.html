WinCom

The W3C libwww Web Commander GUI Tool

WinCom is a sample MS Windows 95/NT GUI application for saving documents remotely using HTTP/1.1. It allows the user to explicitly control the metainformation describing the document to save including the language, type, charset, etc. The Web Commander is part of the the libwww - the W3C Sample Code Library and is available in both source and binary versions.

Binary Installation Guide

Because Windows makefiles are such a pain to handle, I have made an exception and provided a binary version of the Web Commander. In order to install the binary version, complete the following steps:

  1. Download the latest precompiled binary version
  2. Unzip the distribution file into your preferred location. All files in the zip file are packaged under the "bin" folder, so you may want to extract it into C:\, for example. That way, the Web Commander will end up as "C:\bin\WinCom.exe".
  3. There is no installation wizard, but if you want to make a desktop shortcut then you can make a shortcut in the "bin" folder and drag it to the desktop.
  4. If ever you want to uninstall the Web Commander then just delete the "bin" folder (make sure that you don't have anything else there, of course)

Features

  • Uses the libwww for doing HTTP/1.1 PUT
  • Explicit control over metadata describing the source that you are saving like language, charset, etc.
  • Basic support for version conflict detection
  • Support for proxies
