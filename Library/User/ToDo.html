Working List for the Library

This is a list of what's going on in the Library of Common Code. It also is a strong indication of what's going on in the Line Mode Browser.

Plan for Next release

These topics have been implemented and are under test or they are very close to be so. Anyway, you will find it in the next release! You can also look at the Change History to see what's new in the current version.
HTTP/1.0 Compilant HTTP module
An improved HTTP module that supports (almost) all HTTP/1.0 features as currently specified.
PUT and POST
Support for PUT and POST in Library (should have been there a long time ago). This is going to use the multithreaded model which already works for GET and HEAD. It enables the possibility of editing and saving a document to a remote HTTP server and to copy a document from one remote HTTP server to another.
HTTP/MIME-parser
A new HTTP/MIME-parser that can be used as a general module. A new MIME parser has been implemented. It understands all HTTP headers and stores entity information in the anchor structure. Request relevant information is stored in the request structure. See Programmer's Guide for more information.
Content Length Counter Stream
Which is needed for PUT and POST
Windows NT
Better support for Windows is under test
Other
A bunch or minor things...

Under Development

These topics are under development but might not get into the next release:
A better API

Typed Links
This should also have been there a long time ago, but now we want it!
Better support for Accept headers and other headers
Including Accept-Language, etc.
Expires, Last-Modified etc.
As we have a better MIME parser, we can now implement all these nifty features
New News Module
A better News module which also supports PUT and POST.
Full editing capabilities with versioning
So that we can do PUT and POST to a remote server

Next Week's Working Plan

This is what we are going to implement. If somebody should get the idea of writing some of the modules mentioned, it will be appreciated a lot ;-). Contact libwww@w3.org for further coordination.
Access Authentication
Support for various schemes as they pop up (which they tend to do quite a lot these days)
Keep-Alive
Implement support for KEEP-ALIVE requests where the client can GET, PUT, and POST several documents over the same line - all using non-blocking I/O
Whois++
Actually a WhoIs++ module has been implemented (thanks to Michael Mealling, ccoprmm@oit.gatech.edu) in the library but it is not in this release as I haven't found many WhoIs++ servers and that the port chosen is 43 just like the old WhoIs protocol, and that makes it a bit tricky.
README File in directory listings
Make it possible to have both Ascii files (using <PRE>...</PRE>) and HTML files.
Non-blocking DNS interface
Does anyone know of a public domain non-blocking DNS interface to make smart getXbyY() function calls?
OS/2 listings
Implement long FTP directory listings for OS/2 platforms.

Ideas for New Features

This is what we have not started yet but what we would like to implement.
Separation of Protocol Modules
The protocol modules should be separated completely from the HTML machinery so that it is possible to, e.g., get raw FTP directory listings through to the user.

Henrik Frystyk, April 1995, libwww@w3.org
