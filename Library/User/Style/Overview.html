House Style for C Programming

The W3C software modules are all written with a certain house style that almost certainly differs from what you are used to. However, we have our reasons for doing this - well not all of it but most of it! Anyway, the purpose of all W3C code is to be:
  • highly portable so that it works on any thinkable platform
  • consistent in style so that it is easy to read by other project members
Remember, that even it might seem impossible when you are writing the code - somebody will some day have to change it! Therefore, it is very important that we all follow the house style! However, the house style is not cast in stone so if you have good suggestions, do send them - but they really have to be good ;-)

IF YOU DO NOT ABIDE BY THIS GUIDE YOUR CODE WILL ANNOY OTHER TEAM MEMBERS AND MAY NOT PORT.

First of all, please take a look at the existing W3C source code and see what it look like, then some of the things below might make more sense. You will find this Directory structure under /afs/w3.org/hypertext/. The style guide is divided into sections on

  • Modules in straight ANSI C
  • Language 'features'

  • Modules
  • Macros
  • Identifiers
  • Include files
  • Function header
  • Code style
All our code is kept in the CVS code management system, so if you are not familiar with this then please read the online manual.
