Identifiers

When chosing identifier names, it is important that you follow the following rules:
  • Macros should be un upper case entirely unless they mimic and replace a genuine function.
  • External names should be prefixed with HT to avoid confusion with other projects' code. For the rest of the identifier, we use initial capitals a la Objective-C, for example 
    	HTSendBuffer
  • A very simple rule: Do NOT use global variables under any circumstances!
  • Remember to use the PRIVATE and PUBLIC macros to define local or global identifiers
