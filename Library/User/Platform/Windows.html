Notes About Windows

These are the installation and user notes for the Library of Common Code and the Line Mode Browser. Any comments and suggestions are welcome at libwww@info.cern.ch.

As we only have some very old C compilers for the Windows platform, and many of the problems have been fixed in newer version. This port is still in a test phase so please comment! Special thanks to Paul Hounslow, P.M.Hounslow@reading.ac.uk. See also the notes on Windows NT

WinSock DLL
The socket interface is based on a WinSock DLL, preferably a version greater than 1.1.
Dynamic Library (DLL)
The Library can't compile as a DLL out of the box. It requires some modifications and possibly some minor patches. Please remember to send the patches, so that we can put them in!
Direct WAIS access
The normal possibility for linking with the WAIS library is not possible on the Windows NT platform. Therefore the WWW Library module HTWAIS must be excluded in the make file.
Memory Model
The memory model used should be LARGE. This has the limitation that a single data object can't exceed 64K which might cause problems in some situations.
Problem with 16 bit integers
Unfortunately most of the code in the Library is based on 32 bit integers, so there will be overflow in some situations. One solution is to redefine the int as long using a typedef declaration.
