Internals and Programmer's Guide

Any suggestions or ideas are welcome at libwww@w3.org.

This guide describes the modules and their internal relations in the W3C Reference Library. Every module in the Library has a HTML document associated with it containing a detailed description of the functionality and interface to other modules. This page is the top node for the implementation specific documentation and contains links to all the modules in the Library. The documentation is dynamically kept up to date as the actual include files (.h) are generated from the HTML documents using the Line Mode Browser. This document is the last of three documents that each describes different aspects of the Library:

The Architecture
The current document is the first in the series and it describes the architecture of the Library without referring directly to the code itself. This is the "designer view" where bugs are a unknown phenomenon.
The User's Guide
The second document introduces the "official" API of the Library. This document represents the "User's view" and should be sufficient to use the Library without being aware of what's going on underneath the interface. It is also the API that is least exposed to changes. The description contains a set of examples.
The Internals and Programmer's Guide
The last document in the series presents all modules in the Library regardless if they are part of the API described in the User's Guide or not. This is also known as the "hacker's view" of the Library. Each part of the documentation is also available as one big HTML file intended for printout. As a result, the relative links in this document do not work!

When compiling the Library please make sure that you have a relatively new version of the Line Mode Browser that parses the HTML documents correctly (2.13 and newer versions). Find out what version of the Line Mode Browser you are using by typing 
	./www -version
Also remember that when editing the module interfaces or adding functionality then always use the HTML files and not the .h files!
