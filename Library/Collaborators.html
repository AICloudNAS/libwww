Writing directly to the CVS repository is reserved for libwww collaborators who have intimate knowledge of the code base and have proven their ability to use this privilege to the benefit of the libwww user community. You are of course more than welcome (read strongly encouraged) to submit patches to the <www-lib@w3.org> mailing list regardless of whether you have write access or not. If the patch is good, chances are good that it will be added to the CVS codebase by one of the collaborators.

How can I Sign Up?

We don't have a formal mechanism but the idea is that you have to prove your hacking ability. The best way to is to start contributing patches based on the latest code base and to be active on the <www-lib@w3.org> mailing list. Make sure that you have read the libwww style guide before you start.

How can I help?

We have a very interesting set of things to do - your help is required to make timely progress!

HTTP Digest Authentication - in progress by Jose Kahan
Is becoming more and more important the and set of features is extremely interesting. Libwww supports easy addition of multiple authentication schemes - there are already some initial digest stuff.
Regular expressions in URI mapping rules
Libwww already supports URI mappings in the rules module but support for sed like regular expressions would be extremely useful, especially for mirroring done by the Webbot. Libwww already uses regular expressions for handling proxies and other things.
Robot txt file parser - done by John Punin!
The Webbot already knows about the robot META tags - help turning it into a well-behaving robot by understanding robot.txt files as well.
Drag-and-drop  support in Web Commander
Would be cool to have as well as integration with the Explorer menu. It would make it a lot easier to work with
Provide an install shield for the Web Commander
The Web Commander could really use an in install shield to get Windows people up and going. It would be very nice if it could also support installing the libwww DLLs in a suitable location.
Provide better Windows Makefiles
I am waiting (Setember 20, 1998) for autoconf and automake to be released with support for Windows as I am happy about the way this works on Unix. Any better ideas plan for improving Windows makefiles?
Implementing the HTTP Extension Framework
We have many exiting HTTP extensions waiting for the HTTP Extension Framework. Your help is needed! The implementation can be based on the PEP implementation.

Don't forget to send mails to <www-lib@w3.org> about what you are up to or to ask questions!
