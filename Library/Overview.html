Libwww is a general-purpose Web API written in C for Unix and Windows (Win32). With a highly extensible and layered API, it can accommodate many different types of applications including clients, robots, etc. The purpose of Libwww is to provide a sample implementation of HTTP and other Internet protocols and to serve as a testbed for protocol experiments.

March 25, 1998: Libwww Version 5.1l available for HTTP/1.1 testing! Check out the performance!
This release is a "second generation" HTTP/1.1 implementation using persistent connections, pipelining, smart output buffering, and persistent caching. It is the code base that was used in the "Network Performance Effects of HTTP/1.1, CSS1, and PNG" paper
  • This release implements HTTP/1.1 Rev-02!
  • Check out the list of new features
  • Complete HTTP/1.1 compliant client side persistent cache manager
  • Support for zlib based decompression and regular expressions - can do really cool things in the Robot
