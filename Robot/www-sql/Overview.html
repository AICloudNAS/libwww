The Webbot Log Database

This is an example of a web-enabled interface to a MySQL database based on www-sql. The following links provides you with some samples of how you can extract information about broken links, redirections, document titles, last modified dates, content types, link relationships and much much more from the webbot MySQL log database.

When you have set this up, the information is extracted directly from the MySQL database as you go along - neat, huh?

Link Relationships
Here you can find which resources point to which resources - both incoming and outgoing links as well as search for different link types.
Request Information
Here you can find all the information that the libwww robot gathered from the Web requests issued when traversing our site.
Resources Information
Here you can find a lot of metainformation about the resources that we are serving including content type, last modified date, etc.
Broken HTML, Bad Style, etc.
Find out what is wrong about your pages and where the problems are

I use this set of files to check my own links on the W3C website approximately every evening or so and it takes about 50 minutes to check 32,000 links.
