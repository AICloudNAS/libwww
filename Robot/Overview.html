The Libwww Mini Robot

The W3C Mini Robot is a cross platform command line tool for accessing the Web. It is developed in C and serves as an example of a client application using the W3C Sample Code Library and as a test-tool for quick access to the Web from scripts, cron jobs etc. It gives a variety of possibilities for data format conversion, filtering etc. It is primarily intended as a test-tool for quick access to the Web or used in scripts, cron jobs etc.

Features

  • Really fast libwww HTTP/1.1 engine - uses HTTP/1.1 pipelining which speeds up performance a lot
  • Smart use of GET and HEAD - only downloads the strictly necessary
  • Support for regular expressions allows detailed Web traversal constraints
  • Support for traditional Common Log File formats and Referer log file formats
  • A wealth of logging facilities including:
    • Missing or empty ALT tags in inlined images
    • Broken links
    • Document titles based on the TITLE HTML tag and the Title HTTP header field
    • Link relationships between documents based on the LINK HTML tag
    • Modification dates based on the Last-Modified HTTP header field
    • Distributions of content-types and charsets encountered in the traversal

How to Run

Be careful - this is a robot and hence can be used to traverse many links - it should not be used for anything but HTTP/1.1 testing. Especially, it doesn't follow directions indicated in robot.txt files and hence should not be used on the Internet at large.

The robot has a large set of command line options that can be used in a large set of different combinations. You can try and see this simple script in order to see an example of how it can be run.
